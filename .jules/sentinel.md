## 2026-03-24 - SQL Injection in Database Manager
**Vulnerability:** The `get_table_info` method in `src/codomyrmex/database_management/db_manager.py` used string formatting to construct a `PRAGMA table_info` query. While it attempted to escape quotes, string formatting is fundamentally unsafe for dynamic SQL queries and can lead to SQL injection if input is not perfectly sanitized.
**Learning:** PRAGMA statements in SQLite can often be replaced with table-valued functions (e.g., `pragma_table_info()`). These functions can be queried using standard `SELECT` statements (`SELECT * FROM pragma_table_info(?)`), allowing the use of safe, parameterized inputs.
**Prevention:** Always use parameterized queries for dynamic inputs in SQL. When dealing with database metadata or specialized commands (like PRAGMA), investigate if the database driver or engine provides a parameterized alternative (like table-valued functions) before falling back to manual string sanitization or formatting.
