## 2024-03-01 - Command Injection Prevention in operating_system Module
**Vulnerability:** The `operating_system` module (`base.py`, `linux/provider.py`, `mac/provider.py`, `windows/provider.py`) executed native commands via `subprocess.run` with `shell=True`, which allows command injection. Several commands heavily utilized shell pipelines (e.g., `cmd1 | cmd2`).
**Learning:** Fixing `shell=True` requires replacing string commands with lists of arguments using `shlex.split()`. However, `subprocess.run(shell=False)` does not support shell features like pipes (`|`) or output redirection (`>`). Therefore, when converting pipeline strings to list arguments for `shell=False`, the pipelines must be rewritten using pure Python logic (e.g., reading files directly, slicing outputs natively) or simulating them via separate `subprocess.run` calls.
**Prevention:** Always use `subprocess.run(..., shell=False)` with a `list[str]`. Avoid shell pipelines; implement data filtering natively in Python.
