## Purpose
Persistence layer for LLM interactions. Stores traces, generation logs, and performance metrics. Provides structured way to debug and analyze LLM interactions with timestamped or ID-based filenames to prevent collisions. Outputs are categorized by type (raw llm_outputs, structured reports, performance metrics).

## Key Classes and Functions

### Output Organization
- **Raw Outputs** (`llm_outputs/`): Raw content generated by LLMs with format `YYYY-MM-DD_HH-MM-SS_{model}_{task_id}.json`
- **Reports** (`reports/`): Structured analysis reports
- **Performance** (`performance/`): Performance metrics and benchmarks
- **Integration** (`integration/`): Integration test results and validation
- **Test Results** (`test_results/`): Test execution results

### Data Schema
- **Prompt**: The full input sent to the model
- **Completion**: The raw text returned
- **Metadata**: Latency, token usage, model version
