## Purpose

Stores security and analysis reports generated from LLM operations and evaluations. Contains structured JSON reports documenting security assessments, input validation, and compliance checks.

## Active Components

- `security_input_length_limits.json` - Input length security analysis
- `SPEC.md` - Directory specification
- `README.md` - Directory documentation

## Report Types

### security_input_length_limits.json
Documents security-related input length constraints:
- Maximum token limits per model
- Buffer overflow prevention measures
- Input sanitization requirements
- Recommended length limits by use case

### Future Reports
- Prompt injection analysis
- Output content safety evaluations
- Data leakage assessments
- Model bias evaluations

## JSON Report Structure
```json
{
  "report_type": "security_input_length_limits",
  "generated_at": "ISO timestamp",
  "version": "1.0.0",
  "environment": {
    "models_tested": ["list of models"],
    "ollama_version": "version"
  },
  "findings": [
    {
      "model": "model_name",
      "max_input_tokens": 4096,
      "recommended_limit": 3500,
      "overflow_behavior": "truncation"
    }
  ],
  "recommendations": ["list of security recommendations"]
}
```

## Operating Contracts

- Reports are immutable once generated
- New assessments create new timestamped reports
- JSON schema validates report structure
- Findings include severity levels where applicable
- Reports reference test methodology used
