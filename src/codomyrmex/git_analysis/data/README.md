# git_analysis/data — Analysis Output Storage

This directory stores the output of `git_analysis` operations — cached analysis
results and generated reports for the codomyrmex repository itself.

## Contents

| File | Description |
|------|-------------|
| `codomyrmex_description.md` | Live analysis output describing the codomyrmex repo structure, generated by running `git_analysis` tools against this repository |

## Purpose

The `data/` directory serves as the persistent output cache for analysis runs.
When the `git_analysis` module runs history analysis or GitNexus graph queries,
structured output can be written here for later reference.

## Generating / Updating Data Files

```bash
# Re-analyze the codomyrmex repo and update codomyrmex_description.md
uv run python -c "
from codomyrmex.git_analysis import GitHistoryAnalyzer
import json, pathlib

analyzer = GitHistoryAnalyzer('.')
stats = analyzer.get_contributor_stats(max_count=50)
history = analyzer.get_commit_history(max_count=100)
print(f'Analyzed {len(stats)} contributors, {len(history)} commits')
"
```

## Data Retention Policy

- Files in `data/` are tracked in git as living documentation.
- Update `codomyrmex_description.md` when running a documentation sprint.
- Do **not** commit large binary blobs or raw git object dumps here.
- GitNexus graph data (`.gitnexus/`) is gitignored and stored at repo root, not here.
