## Purpose

Stores repository library definitions and metadata persistence files. This directory contains the configuration data for repository management including library definitions, auto-generated libraries from GitHub API, and JSON metadata for tracking repository state.

## Active Components

- `repository_library.txt` - Main repository library definition file with categorized repositories
- `docxology_repository_library.txt` - User-specific repository library for docxology repositories
- `auto_generated_library.txt` - Automatically generated library from GitHub API
- `repository_metadata.json` - Persistent JSON storage for comprehensive repository metadata
- `README.md` - Directory documentation

## Data File Formats

### Repository Library Format (.txt)
```
# Format: TYPE|OWNER|REPO_NAME|URL|DESCRIPTION|LOCAL_PATH_SUGGESTION
OWN|username|repo-name|https://github.com/username/repo-name.git|Description|username/repo-name
FORK|username|forked-repo|https://github.com/username/forked-repo.git|Description|forks/forked-repo
USE|external|library|https://github.com/external/library.git|Description|external/library
```

### Repository Types
- **OWN** - Original repositories for development and contributions
- **FORK** - Forked repositories for contributions to upstream
- **USE** - External repositories for reference/usage (read-only)

### Metadata JSON Structure
```json
{
  "owner/repo-name": {
    "full_name": "owner/repo-name",
    "clone_status": "CLONED",
    "local_path": "/path/to/repo",
    "clone_date": "ISO timestamp",
    "last_sync_date": "ISO timestamp",
    "stats": { "stars": 0, "forks": 0 }
  }
}
```

## Operating Contracts

- Library files use pipe-delimited format with 6 fields per entry
- Lines starting with # are treated as comments
- Empty lines are ignored during parsing
- Metadata JSON is automatically updated by RepositoryManager operations
- Auto-generated libraries should not be manually edited
